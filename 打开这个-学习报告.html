学习框架解决了什么问题？
为什么使用框架？
    框架作用
  1. 系统。比如Bootstrap框架是目前最受欢迎的前端框架。Bootstrap 是基于 HTML、CSS、JavaScript的，它简洁灵活，使得 Web 开发更加快捷。
  2. 直接。直接调用就可使用。无论是布局、css，还是js，都可以直接调用。
  3. 高效。不用自己写代码，并且运行快速。
  4. 兼容。框架发展时间长，自己写的代码没有框架兼容性好。
  5. 成本低。使用框架，开发时间大大缩短，而且框架免费。

三大框架优缺点对比

下面这些是我到现在勉强能理解的各大框架信息，再深入了解需要更多的学习。

  • Vue
      • 我的理解：小而灵活，插件化，对新手友好；但不宜开发大型项目，不够丰富。

      Vue.js 是用于构建交互式的 Web 界面的库。它提供了 MVVM 数据绑定和一个可组合的组件系统，具有简单、灵活的 API。大多程序员在学习新框架的时候都会先从Vue开始。

      特性
    • 易学：官方文档很清晰，比Angular简单易学。
    • 轻量级的框架
      不带有侵略性API,对容器没有依赖性,易于进行配置,易于通用,启动时间较短。 相应的，它的处理能力便有所减弱（如事务功能弱、不具备分布式处理能力），比较适用于开发中小型企业应用。
    • 双向数据绑定
      • 单向数据流：简单的单向数据流指用户访问View，View发出用户交互的Action，在Action里对state进行相应更新。 state更新后会触发View更新页面的过程。这样数据总是清晰的单向进行流动，便于维护并且可以预测。
      • 双向数据流：数据（state）和视图（View）之间的双向绑定。
    • 插件化
    • 不够成熟，影响度也不大
      Vue.js是一个新的项目，没有angular那么成熟。有关于Vue.js多样性/丰富性少于其他一些有名的库。
  • React
      • 我的理解：支持大型项目，非常灵活，门槛稍高。

    • 声明式设计：React采用声明范式，可以轻松描述应用。
      • 声明式设计：提出命令（用封装）
      • 命令式设计：执行命令（写接口很麻烦，所以产生了封装）
    • 高效：
      React通过对DOM的模拟，最大限度地减少与DOM的交互，相应的，速度也很快。
      • DOM：这个还不了解。DOM 定义了访问 HTML 和 XML 文档的标准： “W3C 文档对象模型 （DOM） 是中立于平台和语言的接口，它允许程序和脚本动态地访问和更新文档的内容、结构和样式。”
    • 灵活：
      React可以与已知的库或框架很好地配合；且跨浏览器兼容，标准化的API，在IE8中都是没问题的。
    • 模块化：
      为程序编写独立的模块化UI组件，这样当某个或某些组件出现问题是，可以方便地进行隔离。
    • 单向数据流：
      • 优点：这样做的好处是， 所有状态变化都可以被记录、跟踪,状态变化通过手动调用通知,源头易追溯,没有“暗箱操作”。 同时组件数据只有唯一的入口和出口,使得程序更直观更容易理解,有利于应用的可维护性
    • 同构、纯粹的javascript：
      因为搜索引擎的爬虫程序依赖的是服务端响应而不是JavaScript的执行，预渲染应用有助于搜索引擎优化。
  • Angular
      • 我的理解：支持大型项目，内容丰富且完善；但是较难深入学习，初始化慢。

      React框架是起源于Facebook的项目，当时在公司内部盛行JavaScript框架，但是感觉不是很满意，就写了React框架。React可以轻易解决跨浏览器兼容的问题，主要通过对DOM的模拟减少与DOM的交互做到。

      React的模块化把组件进行了隔离，出现问题的时候更方便程序员对其进行修改，而且由于JavaScript，因此更有利于搜索引擎的优化。

      特性
    • 模板功能丰富:自带极其丰富的angular指令。
    • 双向数据绑定
      • 优点：
        • 无需和单向数据绑定那样进行 CRUD操作，双向数据绑定最常应用在就表单上， 这样当用户在前端页面完成输入后，不用任何操作，我们就已经拿到了用户输入好的数据，并放到数据模型中了。
    • 自定义指令：自定义指令后可以在项目中多次使用。
    • 坚实基础和支持：angularjs是互联网巨人谷歌开发，这也意味着他有一个坚实的基础和社区支持。
    • 难以学习：文档例子少，深入后概念很多，学习中较难理解。
